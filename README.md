# 三角洲行動：烽火地帶

Minecraft 1.20.1 Forge 硬核撤離射擊模組。

這個專案的目標，是把「摸金、搜刮、撤離、倒地救援、死亡掉落」這一套硬核生存循環搬進 Minecraft，並以可擴充的資料驅動架構去串接武器、醫療、背包、幹員與後續外部模組整合。

## 目前版本能做什麼

目前這個模組已經能完整跑起一個基礎版的撤離玩法閉環。你可以帶裝進圖、搜刮物資、在地圖上打 AI 或其他玩家、去撤離點讀條離開；如果被打倒，也不會立刻結束，而是會先進入倒地狀態，等隊友按住右鍵救援，或你自己靠擊殺敵人恢復行動能力。

簡單說，現在已經有以下核心內容：

- 撤離點與撤離倒數
- 戰術背包與安全箱
- 黑傷 / 流血 / 止痛 / 手術包等醫療系統
- 倒地、救援、反覆倒地懲罰
- 屍體外觀與死亡包
- AI / 玩家死亡掉裝與搜刮掉落
- 戰利品箱與資料表掉落
- 基礎 KubeJS 合成配方

## 核心玩法循環

### 1. 進圖

玩家可以帶著自訂裝備進入地圖，也可以裸裝跑刀。進圖後的重點不是刷怪，而是：

- 搜刮地圖上的高價值物資
- 清掉 AI 士兵或其他玩家
- 找到撤離點並成功撤離

### 2. 搜刮與戰鬥

目前已實作的搜刮來源包含：

- 戰利品箱
- 敵對生物死亡掉落
- 玩家死亡後的屍體包
- 撤離帶出後的安全保存

### 3. 撤離

玩家站上撤離點後會開始倒數。倒數完成後會被傳回主世界出生點，並送出撤離成功訊息。

如果中途離開撤離區，倒數會取消。

## 已實作系統

## 1. 裝備系統

### 物品類型

目前物品已經分成幾個大類：

- 醫療道具
- 彈藥道具
- 裝備道具
- 高價值戰利品
- 武器道具

### 戰術背包與安全箱

這套系統不是原版背包，而是二維網格式的物品管理：

- 物品有寬高概念
- 可以旋轉
- 可以從網格中取出或放入
- 有獨立同步封包
- 安全箱會在玩家死亡時保留
- 普通網格背包會隨死亡流失

### 目前可用物品

目前已註冊的物品包含：

- 止血帶
- 止血繃帶 / Bandage
- 手術包
- 醫療箱 / Medkit
- 止痛藥
- 步槍彈匣
- 手槍彈匣
- 彈藥箱
- 戰術背心
- 戰術背包
- 夜視鏡
- 金條
- 情報資料
- 古代遺物
- 醫療物資
- 突擊步槍
- 衝鋒槍
- 霰彈槍
- 狙擊步槍
- 手槍

## 2. 醫療與健康系統

這是目前最完整的核心系統之一。

### 部位傷害

已經有七大部位健康的概念：

- 頭
- 胸
- 腹部
- 左臂
- 右臂
- 左腿
- 右腿

### 黑傷效果

部位血量歸零後會進入黑傷，並對應不同懲罰：

- 腿黑了會減速
- 手黑了會挖掘 / 使用變慢
- 腹部黑了會更容易飢餓
- 流血會持續掉血

### 可用醫療物品

- 止血帶：處理流血
- 藥包 / Bandage：基礎回血
- 手術包：修復黑傷部位
- 止痛藥：暫時抑制黑傷懲罰
- 醫療箱：較高階的回血手段

### 目前行為

- 使用止痛藥後可暫時壓住黑傷副作用
- 手術包可把黑傷部位修回可用狀態
- 流血狀態會定時扣血
- 撞傷 / 枪傷可一路往死亡或倒地推進

## 3. 倒地與救援系統

這一套是目前的重點功能。

### 倒地規則

當玩家受到致命傷時，不會直接結束，而是先進入倒地：

- 60 秒基礎倒地倒數
- 倒地時不能攻擊、不能使用物品
- 倒地時會持續受限，視覺與操作都會被壓住

### 救援規則

- 隊友必須按住右鍵才能持續救援
- 鬆手會立刻中斷
- 救援重來要重新讀條
- 救援成功後會把玩家拉回來並恢復部分血量

### 反覆倒地懲罰

目前已加入反覆倒地加重機制：

- 每次再倒地，後續倒地時間會被壓縮
- 反覆倒地的救援壓力也會增加
- 讓玩家不能無腦硬衝，必須重視撤退與支援

### 自救規則

如果玩家在倒地期間擊殺敵人，會觸發自救恢復，重新回到可行動狀態。

## 4. 屍體與死亡包

玩家死亡後，現在不只是單純掉物品。

### 屍體外觀

死亡時會在死亡地點生成一個屍體標記，作為死亡位置的視覺提示。

### 死亡包 / Corpse Bag

死亡後的物資會被收進一個可撿取的屍袋方塊中：

- 玩家死亡掉落會集中進屍袋
- 玩家身上的網格背包內容會進去
- 安全箱內容會依保留邏輯留下
- 其他人可以右鍵撿取屍袋

這讓死亡不只是純懲罰，也會形成可回收戰利品的風險點。

## 5. 搜刮機制

### 戰利品箱

目前已經有三種等級的戰利品箱：

- 普通
- 稀有
- 史詩

戰利品箱內容是從 JSON loot table 讀取，不是硬編寫死。

### 掉落來源

現有掉落來源包含：

- 戰利品箱
- 敵對生物死亡掉落
- 屍體包
- 地圖上的物資點

### 資料驅動優點

這樣的好處是後續要調整掉落，不需要一直改 Java：

- 改 loot table 就能換內容
- 可以很快做事件掉落
- 可以做普通 / 高階 / 首領差異化

## 6. AI 與戰鬥掉落

目前已經有基礎的敵對生物死亡掉落邏輯：

- 敌對生物死後會掉出裝備與戰術物資
- 會優先保留主手、副手、護甲等內容
- 也會隨機掉出醫療、彈匣、背包、高價值物資

這是未來接 TaCZ / 戰術 AI 模組時的基礎接口。

## 7. 撤離系統

### 撤離點

- 站上撤離點會開始倒數
- 離開撤離區會中止
- 倒數結束會成功撤離

### 撤離成功效果

- 玩家會被傳送回主世界出生點
- 成功訊息會顯示在聊天欄
- 安全箱物資保留

## 8. 幹員資料系統

目前幹員資料架構已存在，包含：

- 幹員選擇資料
- 幹員能力資料結構
- 玩家重生複製時的能力保存

這一部分目前比較偏資料層和擴充層，尚未完整接成遊戲內選單或完整技能樹，但已經把資料結構先建立好。

## 操作方式

### 現在可用的操作

- 開啟戰術背包
- 開啟安全箱
- 在網格中放入、取出、旋轉物品
- 右鍵使用醫療物品
- 倒地時等待隊友按住右鍵救援
- 撤離點等待讀條成功撤離

### 背包操作

- 左鍵點選物品取出
- 對網格右鍵 / 控制項處理放置與同步
- `R` 可旋轉支援旋轉的物品

## 資料與內容檔案

這個專案有不少內容是資料驅動的：

- `src/main/resources/data/...`：loot table
- `src/main/resources/assets/...`：語言、模型、方塊狀態
- `kubejs/server_scripts/...`：基礎合成配方

也就是說，很多未來擴充可以直接靠 JSON / KubeJS 完成，不一定要先寫 Java。

## 建置與執行

### 本機建置

```bash
./gradlew build
```

Windows 可以用：

```bat
gradlew.bat build
```

### 開發啟動

```bash
./gradlew runClient
```

### 資料生成

```bash
./gradlew runData
```

## 目前專案狀態

目前這個 mod 已經不是單純的骨架，而是有以下可用成果：

- 可進行撤離玩法
- 可進行死亡後救援
- 可做基礎搜刮與物資保存
- 可用醫療道具處理黑傷與流血
- 可撿屍體包
- 可透過 loot table 調整掉落

## 後續最適合接的內容

如果要繼續往完整「烽火地帶」靠攏，下一步最適合接的是：

- 地圖內固定搜刮點類型細分
- AI 士兵與首領掉落表
- 護甲 / 頭盔 / 背包的更細重量與防護差異
- 幹員技能與組隊定位
- 與外部槍械模組的更深度整合
